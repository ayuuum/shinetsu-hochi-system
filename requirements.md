## 1. 本書の目的

本書は、株式会社信越報知（以下「甲」）が導入する「社員・資格管理システム」について、株式会社Amber（以下「乙」）が開発する機能の範囲・仕様・品質基準を定義し、甲乙間の合意事項として確定するものである。

---

## 2. システム概要

### 2.1 背景と課題

株式会社信越報知は、消防設備・セキュリティ設備・通信設備の設計・施工・メンテナンスを主力事業とし、約20名の専門技術者を擁する。現在、以下の課題を抱えている。

| # | 現状の課題 | 影響 |
|---|---|---|
| 1 | 社員情報・資格情報がExcelや紙で分散管理 | 情報検索に5〜10分を要する |
| 2 | 消防設備士の講習期限管理が属人的 | 年2〜3件の受講漏れリスク |
| 3 | 施工実績書の作成が手作業 | 1件あたり30分〜1時間のコスト |
| 4 | 車検・保険期限の管理が散在 | 更新漏れリスク |
| 5 | アルコールチェックが365 Excelで手入力 | 現場負担が大きく記録漏れの懸念 |

### 2.2 導入目的

上記課題を解決し、資格管理業務を一元化・自動化することで、技術者が本業（現場での安全管理）に集中できる環境を構築する。

### 2.3 達成指標

| 指標 | 現状 | 目標値 |
|---|---|---|
| 資格講習の受講漏れ | 年2〜3件リスク | **年0件** |
| 社員情報の検索時間 | 5〜10分 | **10秒以内** |
| 施工実績書の作成時間 | 30分〜1時間/件 | **5分以内** |
| データ最新性 | 月1回更新 | **リアルタイム** |

## 4. 機能要件

### 4.1 ダッシュボード【REQ-001】

| 要件ID | 要件内容 |
|---|---|
| REQ-001-01 | サマリーカード（社員総数、期限切れ件数、30日以内件数、資格総数）を表示する |
| REQ-001-02 | 要対応アラート一覧を期限切れ→緊急→警告の優先度順で表示する |
| REQ-001-03 | 各アラートに社員名・所属・資格名・残日数を表示する |
| REQ-001-04 | 今月の予定（健康診断・車検・点検等）を一覧表示する |
| REQ-001-05 | アルコールチェック未記録者をダッシュボードに警告表示する |

---

### 4.2 社員台帳管理【REQ-002】

| 要件ID | 要件内容 |
|---|---|
| REQ-002-01 | 社員の新規登録・編集・削除（論理削除）ができる |
| REQ-002-02 | 基本情報（社員番号・氏名・フリガナ・生年月日・性別・住所・電話番号・メール）を管理する |
| REQ-002-03 | 雇用情報（入社日・退社日・雇用形態・所属・職種・役職）を管理する |
| REQ-002-04 | 保険情報（雇用保険番号・健康保険番号・厚生年金番号）を管理する |
| REQ-002-05 | 緊急連絡先（氏名・続柄・電話番号）を管理する |
| REQ-002-06 | 顔写真をアップロード・表示できる |
| REQ-002-07 | フリーワード検索（氏名・フリガナ）ができる |
| REQ-002-08 | フィルタ検索（所属拠点・資格種別・雇用形態）ができる |
| REQ-002-09 | 社員一覧をCSVエクスポートできる |

---

### 4.3 資格・免許管理【REQ-003】

| 要件ID | 要件内容 |
|---|---|
| REQ-003-01 | 資格種別マスタ（消防設備士・電気工事士等 25種プリセット）を初期搭載する |
| REQ-003-02 | ユーザーが資格種別をカスタム追加できる |
| REQ-003-03 | 社員ごとに資格の取得日・証書番号・交付機関・取得区分を登録できる |
| REQ-003-04 | 資格証書の画像を複数枚アップロードできる |
| REQ-003-05 | 講習受講履歴（日付・種別・会場）を記録できる |
| REQ-003-06 | 講習申込ステータス（未着手/申込済/受講済）を管理できる |
| REQ-003-07 | 特定の資格保持者一覧を抽出できる（例：「甲種4類」かつ「本社所属」） |

#### 消防設備士 講習期限 自動算出【REQ-003-A】

| 要件ID | 要件内容 |
|---|---|
| REQ-003-A01 | 初回講習期限：免状交付日以降の最初の4/1を起算日とし、起算日＋2年を期限とする |
| REQ-003-A02 | 2回目以降：前回講習日以降の最初の4/1を起算日とし、起算日＋5年を期限とする |
| REQ-003-A03 | 免状写真更新期限：交付日から10年ごとを期限とする |
| REQ-003-A04 | 講習受講記録の追加時に、次回期限を自動再計算する |

---

### 4.4 期限アラート表示【REQ-004】

| 要件ID | 要件内容 |
|---|---|
| REQ-004-01 | 画面表示時に期限付きレコードを確認し、ダッシュボード・一覧にアラート表示する |
| REQ-004-02 | 4段階のアラートレベルで判定する |

**アラートレベル定義：**

| レベル | トリガー | 通知方法 |
|---|---|---|
| INFO（🔵） | 期限60日前 | ダッシュボードのみ |
| WARNING（🟡） | 期限30日前 | ダッシュボードのみ |
| URGENT（🟠） | 期限14日前 | ダッシュボードのみ |
| CRITICAL（🔴） | 期限切れ | ダッシュボードのみ |

| 要件ID | 要件内容 |
|---|---|
| REQ-004-03 | メール通知は運用対象外とし、期限確認は画面内アラートで行う |
| REQ-004-04 | アラート表示は期限切れ・14日以内・30日以内・60日以内で識別できる |
| REQ-004-05 | アラート対象：資格講習期限、車検期限、自賠責/任意保険期限、健康診断期限 |

## 5. 非機能要件

### 5.1 性能要件【NFR-001】

| 要件ID | 要件内容 |
|---|---|
| NFR-001-01 | 画面表示は2秒以内に完了する |
| NFR-001-02 | 検索結果は1秒以内に返却する |
| NFR-001-03 | 同時接続ユーザー数：最大10名 |

### 5.2 セキュリティ要件【NFR-002】

| 要件ID | 要件内容 |
|---|---|
| NFR-002-01 | 全通信をHTTPS（TLS 1.2以上）で暗号化する |
| NFR-002-02 | 個人情報に対してSupabase Row Level Security (RLS) を適用する |
| NFR-002-03 | パスワードはハッシュ化して保存する（Supabase Auth標準） |

### 5.3 可用性・バックアップ【NFR-003】

| 要件ID | 要件内容 |
|---|---|
| NFR-003-01 | データベースは日次自動バックアップ（30日間保持）を行う |
| NFR-003-02 | SaaS基盤（Supabase/Vercel）の可用性に準拠する |

### 5.4 対応環境【NFR-004】

| 要件ID | 要件内容 |
|---|---|
| NFR-004-01 | ブラウザ：Chrome / Safari / Edge の最新版に対応する |
| NFR-004-02 | iOS / Android のスマートフォンブラウザに対応する（レスポンシブデザイン） |
